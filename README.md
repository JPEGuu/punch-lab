# 勉強用リポジトリ

## 1. プロジェクト概要

- **目的**: 従業員の勤怠打刻と、管理者の集計業務の効率化。
- **ターゲット**:
  - 従業員（営業職メイン）: 外出先からスマートフォンで利用。
  - 管理者（事務職）: 社内デスクトップPCで利用。
- **形態**: レスポンシブWebアプリケーション

## 2. 機能要件

### 2.1 全ユーザー共通

- **ログイン**: ユーザーIDとパスワードで認証を行う。
- **ログアウト**: セッションを破棄して終了する。
- **打刻機能**: 以下の4種類を記録可能とする。
  - 出勤
  - 退勤
  - 休憩開始
  - 休憩終了
- **履歴確認**: 自分の当月の勤務状況（打刻ログ）を一覧表示する。

### 2.2 管理者専用機能

- **社員一覧**: 全従業員のリストを表示する。
- **勤怠修正**: 従業員の打刻漏れや間違いを修正・削除する。
- **月次集計**: 社員ごとの「月間総労働時間」を自動計算して表示する。
- **CSV出力**: 給与計算ソフト連携用に、勤怠データをCSV形式でダウンロードする。

## 3. 技術スタック・非機能要件

- **言語**: Go
- **データベース**: SQLite (ファイルベースRDB)
- **認証**:
  - セッション管理 (Cookie利用)
  - パスワード保存: bcryptによるハッシュ化
- **UI**: HTMLテンプレート (サーバーサイドレンダリング) ※シンプル構成

## 4. データベース設計 (Schema)

### 4.1 Users テーブル (従業員情報)

| カラム名 | データ型 (SQLite) | 説明 |
| :--- | :--- | :--- |
| `id` | INTEGER | 主キー, 自動連番 |
| `name` | TEXT | ログインID・表示名 (UNIQUE) |
| `password_hash` | TEXT | bcryptハッシュ化済みパスワード |
| `role` | INTEGER | 権限 (0:一般 / 1:管理者) |

### 4.2 Logs テーブル (打刻実績)

| カラム名 | データ型 (SQLite) | 説明 |
| :--- | :--- | :--- |
| `id` | INTEGER | 主キー, 自動連番 |
| `user_id` | INTEGER | `users.id` への外部キー |
| `time` | TEXT/DATETIME | 打刻日時 (ISO8601形式推奨) |
| `type` | INTEGER | 打刻種別 (1:出勤, 2:退勤, 3:休憩開始, 4:休憩終了) |

## 5. 重要ロジック・仕様

### 5.1 日またぎの計算

- **ルール**: 勤務実績は「出勤打刻を行った日の実績」として計上する。
- **例**: 10/1 22:00 出勤 ～ 10/2 05:00 退勤 → 全て「10/1」の勤務とする。

### 5.2 労働時間の計算ロジック (Goプログラム側)

1. 対象期間のログを時系列順（昇順）で取得する。
2. ログをループ処理し、変数を活用して状態（出勤中、休憩中など）を保持する。
3. 「退勤 - 出勤」から「休憩合計」を引いて実働時間を算出する。
